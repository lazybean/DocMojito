The mojit collection

How to make your mojit usable by other people/app.

Introduction

You are now a crazy bartender, having created many delicious mojit and blend them in as many refreshing mojito. You now want to be able to reuse any of your mojit in any of your new project.

You will learn an easy way to make 'local' packages from your mojit and use them in your apps, thank to npm link

Steps

Catch them all

Gather all your mojit together in a folder, they do not need to stay in your apps anymore.

Each mojit needs to be a npm package. You need to add a 'package.json' file in each mojit folder

 
{
  "name": "myMojitName",
  "description": "The mojit everybody want is his mojito",
  "version": "0.1.0",
  "author": {
    "name": "myName",
    "email": "myemail@mymail"
  },
  "engines": {
    "node": "> 0.6"
  },
"yahoo": {
  "mojito": {
    "version": "> 0.6",
    "type":  "mojit"
  }
},
  "keywords": ['mojit']
}

The important things are, the "name" property that has to be identical to the mojit name, and the yahoo properties (please notice the "mojit" type.

Bind them

With the following simple command, you mojit (package) can be globally symlinked, becoming available system wide.

 
npm link

Use them

In you mojito app, you can create a symlink to any of your mojits with:

 
npm link myMojitName
